# CRO Patterns

These patterns are tuned for early-stage product marketing and signup funnels.

## Pattern 1: Audit the commitment, not the page in isolation

Always identify:

- traffic source
- target action
- form of commitment
- handoff after conversion

The page is only one step in that chain.

## Pattern 2: Four recurring failure buckets

- **Hierarchy** — weak H1, unclear primary CTA, competing actions
- **Proof / trust** — no social proof, no evidence, no safety signals
- **Friction** — too many fields, dead-end flows, awkward mobile behavior
- **Measurement** — broken event logging, no attribution, no activation definition

## Pattern 3: Above-the-fold must answer four questions

Within the first screen, the user should be able to tell:

- what this is
- why it is different
- what to do next
- why they should trust it

## Pattern 4: Signup is not activation

Audit the handoff from:

- click → signup
- signup → first value
- first value → paid or retained behavior

If the surface optimizes only top-of-funnel conversion, call that out.

## Pattern 5: Every experiment needs a metric and a kill rule

Default fields:

- hypothesis
- primary metric
- guardrail metric
- minimum sample / date window
- kill threshold

## Pattern 6: Fix ranking should separate quick wins from structural blockers

Useful buckets:

- `quick win`
- `medium lift`
- `structural`
- `blocked by measurement`

## Pattern 7: Services and platform pages need a value stack

When the page is selling Zerg more broadly than one product, the first screen should usually communicate:

- the hard problem you solve
- the implementation model
- the economic upside
- the system layer that makes it credible

For this class of page, strong candidates include:

- custom-tailored solutions rather than generic consulting
- software-cost compression relative to the current SaaS stack
- robust shared data infrastructure instead of brittle integrations
- AI chat / agent capability as an underlying operating layer

Audit whether the page says only "we build things" or whether it actually names the new steady state the buyer gets.
