# Launch Package Template

Use this for production-bound product launch videos.

If the launch needs both depth and distribution, use the launch video system structure below instead of treating the project as one master video with resized exports.

## 1. Creative Direction

- Video type
- Runtime
- Channel
- Thesis
- Viewer state before watching
- Viewer action after watching
- Core mechanic: the single product transformation the video is built around

## 2. Message Hierarchy

1. Primary promise
2. Proof moment
3. Supporting use case
4. Trust signal
5. CTA

## 3. Timed Script

| Time | Audio / Dialogue | On-screen Text | Visual | Notes |
|---|---|---|---|---|

Include two alternate hooks when the video is for social, paid, Product Hunt, or homepage hero use.

## 4. Shot Sequence

| Time | Shot | Source | Edit Intent | Production Notes |
|---|---|---|---|---|

## 5. Storyboard Outline

|---|---|---|---|---|---|

## 6. Production Plan

| Item | Type | Owner | Source / Location | Due | Status | Notes |
|---|---|---|---|---|---|---|

Include screen states, product data, logo files, customer approvals, music, voiceover, and captions.

## 7. Edit Direction

- Pacing
- Music / sound design
- Caption style
- Motion treatment
- End card
- Cutdown plan
- Thumbnail or first-frame direction

## 8. Variant Plan

|---|---|---|---|---|

## 9. Approval And Risk List

- Product accuracy
- Legal or claims
- Customer/logo use
- Security/privacy
- Brand
- Accessibility and captions

## Launch Video System Structure

Use this when planning one longer video plus multiple short clips.

### Anchor Video

Target 75-100 seconds. Purpose: explain the launch, make the product believable, and give the campaign a canonical artifact.

| Time | Beat | Script / Audio | Visual Proof | Notes |
|---|---|---|---|---|
| 0-5s | Hook |  |  | Must state or show the core change. |
| 5-15s | Problem / old way |  |  | Make the pain concrete. |
| 15-25s | Reveal |  |  | Name product/feature plainly. |
| 25-55s | Core proof |  |  | Show the hardest-to-believe workflow. |
| 55-75s | Use cases / expansion |  |  | Add only the strongest adjacent cases. |
| 75-90s | Trust + CTA |  |  | Customer, metric, founder, technical proof, or clear next action. |

### Modular Clip Slate

Plan 3-8 clips. Target 20-30 seconds each.

Use `clip-slate-generator.md` to choose which clips belong in the slate.

| Clip | Job | Audience / Channel | Hook | Proof Moment | CTA | Source From Anchor? |
|---|---|---|---|---|---|---|

### Clip Types

- Pain clip: dramatize the old workflow and the new way.
- Proof clip: one product action with visible result.
- Use-case clip: one audience or workflow.
- Objection clip: security, quality, setup time, accuracy, cost, or trust.
- Founder clip: why this exists now.
- Customer/proof clip: quote, result, logo, or benchmark.
- Product Hunt/community clip: fast "what it does" scan.

### Production Rule

Capture for the system, not only for the anchor video. Record extra clean product states, vertical-safe crops, isolated proof moments, CTA cards, and alternate hooks while production is already set up.
