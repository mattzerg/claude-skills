# Brand Guide Creator

Use this skill when Matt asks for a brand guide, brand system, design rules, visual language, logo usage guidance, or a refresh of existing brand docs.

## Workflow

1. Read the brand repository first if it exists: `Brand/README.md`, `Brand/asset-catalog.md`, `Brand/tokens/`, and existing `Brand/brand-guide.md`.
2. Ground every rule in a source: live-site CSS/components, logo SVGs, branded collateral, existing style notes, or repeated approved assets.
3. Write the guide for practical use, not brand theater. Cover:
   - brand snapshot
   - logo usage
   - colors and accessibility notes
   - typography
   - layout principles
   - imagery rules
   - collateral/export guidance
   - gaps / open decisions
4. Keep unsupported claims out. If a design source is missing, say so.
5. Add machine-readable token updates only when the guide changes a token.

## Zerg-Specific Defaults

When working in the Zerg vault, prefer the current Zerg brand repository at `MattZerg/Brand/`. The working visual language is cream paper, charcoal structure, burnt-orange emphasis, restrained green accents, Space Grotesk, Space Mono, sharp rules, and product-led visuals.

## Quality Bar

The guide should help someone make the next landing page, one-pager, social card, or launch asset without reverse-engineering old files. Keep it direct, concrete, and tied to actual assets.
