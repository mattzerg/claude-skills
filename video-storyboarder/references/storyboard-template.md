# Storyboard Template

> **For measured numbers** (text sizing % of frame height, hold=reading-time, frame-content mix, safe-area) and **defined jargon** (focal subject, proof invariant), use `storyboard-techniques.md` — it supersedes the field sketches below wherever a measured value exists. This file is the field catalog; the techniques catalog is the measured contract.

## Frame Fields

- Frame number
- Time range
- Purpose of the frame
- Story role: tension, reveal, proof, escalation, outcome, CTA
- Composition
- Focal subject: the one thing the viewer should look at first
- Depth plan: foreground, midground, background
- Subject / UI state
- Camera or screen movement
- On-screen text
- Audio or dialogue
- Transition in/out
- Required assets
- Proof invariant: what must be true at the end of this frame
- Aspect-ratio notes
- Risk or dependency

## Fidelity Levels

### Text Board

Use for early concepting. One compact row per frame.

### Production Board

Use when another person will film, record, design, or animate from the board. Include exact assets, camera notes, UI states, and timing.

For stylized software UI, include exact source and destination states. Example: card starts in column B slot 1, travels to column C slot 2, source count changes 3→2, destination count changes 1→2, feed row appears only after the landing.

Also include the composition intent. A board frame is not enough. Specify whether the shot is a hero surface, macro proof crop, split causal chain, exploded UI layer, montage tile, or end card.

### Animatic Notes

Use when pacing and movement matter more than art. Include frame duration, motion direction, hold length, and transition.

### Prompt Board

Use when generating temporary thumbnails or concept art. Include visual style, subject, composition, lighting, aspect ratio, and negative constraints.
