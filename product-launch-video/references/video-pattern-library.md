# Video Pattern Library

Use this when the video needs stronger taste, sharper openings, or clearer visual mechanics.

## Hook Patterns

- **Result First**: Start with the final output, then show how it happened.
- **Old Way / New Way**: Split or sequence the painful workflow against the product-enabled workflow.
- **Impossible Prompt**: Show an input that sounds too ambitious, then reveal the output.
- **Timer / Speedrun**: Compress time to make speed tangible.
- **Hidden Cost**: Expose the waste, risk, or complexity the audience has normalized.
- **Founder Claim**: A direct line from a credible human, immediately supported by product proof.
- **Customer Constraint**: Start with a real user constraint and show the new path.

## Proof Patterns

- Before/after UI state
- Realistic seeded data
- Side-by-side workflow comparison
- Output artifact reveal
- Customer quote over product action
- Benchmark or metric card with source note
- Cursor path plus zoomed result
- Human reaction only after the product proof is visible

## UI Treatment Patterns

- Crop to the state change; do not show full desktop UI when the important object is small.
- Use zooms to explain hierarchy, not as decoration.
- Smooth cursor movement and remove dead travel.
- Use callouts for one thing at a time.
- Use realistic data; placeholder data weakens proof.
- Show inputs and outputs. Avoid long middle states unless they build trust.

## Ending Patterns

- Product name + one-line promise + CTA
- Result artifact held on screen + CTA
- Founder or customer final line + product proof end card
- Loop back to first frame for silent homepage/social loops

## Anti-Patterns

- Launch starts with "We're excited to announce..."
- Feature inventory without a controlling idea
- Abstract benefit copy over decorative motion
- Tiny UI that cannot be read on mobile
- CTA appears only in narration
- Claims without visible proof or a marked source need
- End card too short to read
