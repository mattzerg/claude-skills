# Edit Patterns

> **For measured numbers** (pacing/MSL by format, transition grammar, music-out + end-card silence lengths, mix levels, silence-trim), use `edit-techniques.md` — it supersedes the prose below wherever a measured value exists. These patterns give the editorial shape; the techniques catalog gives the post numbers.

## Product Launch

Fast hook, product reveal, proof beats, short expansion, CTA. Avoid feature-tour sprawl. Hold the end card long enough to read.

Editor stance:

- Cut on story turns and visible state changes, not on implementation milestones.
- If the clip is a software launch, require proof by 5-10 seconds and a visible change every 2-4 seconds.
- Use hard cuts, camera-state changes, or title cards to create rhythm; do not leave a single passive UI hold to carry the whole edit.
- Keep UI proof shots readable: usually 3-4 seconds, cropped to the active state.
- Reject edits where the board, feed, counts, chat, or final UI state disagree.
- Reject UI sequences where multiple consecutive shots use the same flat full-screen composition. Change shot size or composition mode: hero surface, macro proof, split causal chain, exploded UI layer, montage tile, end card.

## Product Demo

Cut around state changes, not clicks. Use zooms/crops for dense UI. Remove dead cursor travel and loading unless they build trust.

For staged UI, preserve proof continuity: source state, input, motion, output state, and verification should feel like one causal chain.

Badly composed UI cannot be fixed by faster cutting. First choose the focal subject and crop, then cut.

## Social Short

First-frame clarity, rapid proof, captions, and a clean loop or hard CTA. Assume sound-off and platform overlays.

## Paid Ad

Use multiple hooks, early proof, clear problem/solution contrast, and aggressive trimming. Keep the CTA visually present near the end.

## Founder Video

Protect authenticity but remove throat-clearing. Use b-roll to cover edits and keep visual energy. Preserve the strongest exact line.

## Customer Story

Paper-edit around the customer quote spine. Use company narration only to clarify context, not to overwrite the customer's proof.

## Webinar / Event Cutdown

Find self-contained moments. Add context cards where the excerpt would otherwise feel abrupt. Remove housekeeping and long setup.
